# Contributing to Herbst UI

Thanks for your interest! Herbst UI follows an **issue-first workflow**:

1. **Open an issue** describing the bug or proposal before writing any code.
2. Wait for the issue to be discussed and accepted (`approved` label).
3. Fork, create a branch, and open a **Pull Request linked to the issue** (`Closes #123`).
4. Every PR requires maintainer review — `main` is protected and accepts no direct pushes.

PRs without a linked, accepted issue will be closed.

The full guide (setup, code standards, what to contribute) lives in the docs: `/pt/docs/contributing` · `/en/docs/contributing`.

Locally: `npm install && npm start` → `http://localhost:4200/en/docs/contributing`.
